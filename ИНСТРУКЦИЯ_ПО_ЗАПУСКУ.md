## 1. Описание проекта

**FreshLife** — это веб-приложение для онлайн-доставки продуктов питания, реализующее полный цикл работы с каталогом товаров:

- просмотр каталога с фильтрацией по категориям, сортировкой и поиском;
- управление товарами через административный интерфейс (CRUD — создание, просмотр, редактирование, удаление);
- загрузка изображений через форму или скрипт массовой загрузки;
- адаптивная вёрстка (корректно отображается на мобильных устройствах, планшетах и ПК);
- поддержка светлой и тёмной темы.

**База данных:** SQLite — не требует установки отдельного сервера БД, файл `prisma/dev.db` создаётся автоматически.

## 3. Быстрый запуск (5 команд)

Откройте терминал (PowerShell или cmd) **в папке проекта** и выполните команды по порядку:

```bash
# 1. Установка зависимостей
npm install

# 2. Создание / обновление схемы базы данных
npm run db:push

# 3. Заполнение БД тестовыми данными (15 товаров, 5 категорий)
npm run db:seed

# 4. Запуск сервера разработки
npm run dev
```

После этого откройте браузер по адресу:

> **http://localhost:4000**

## 4. Что вы увидите в браузере

### Главная страница — каталог товаров (`/`)

- **15 товаров** в **5 категориях**: Фрукты и овощи, Молочные продукты, Мясо и птица, Напитки, Хлеб и выпечка.
- **Фильтрация** по категории через выпадающий список.
- **Поиск** по названию товара (с задержкой debounce 300 мс, чтобы не посылать запрос на каждую букву).
- **Сортировка** по цене (возрастание / убывание) и по названию.
- **Пагинация** — по 12 товаров на странице с плавной прокруткой вверх.
- Счётчик «Показано X–Y из Z товаров».

### Страница администрирования — управление товарами (`/admin`)

- Таблица всех товаров с кнопками «Редактировать» и «Удалить».
- Кнопка «+ Добавить товар» — открывает модальную форму.
- **Форма создания/редактирования:** название, описание, цена, остаток, единица измерения, категория, загрузка изображения.
- Удаление товара с подтверждением.
- Пагинация в таблице (по 10 товаров на странице).

### Переключатель темы

Кнопка в шапке сайта переключает между светлой и тёмной темой. Выбор сохраняется в `localStorage`.

## 7. Массовая загрузка изображений (bulk-upload)

Скрипт позволяет одной командой загрузить изображения сразу для нескольких товаров, не нажимая кнопки загрузки в интерфейсе для каждого товара вручную.

### Как это работает

1. Вы кладёте файлы изображений в папку `scripts/images/`.
2. Имя файла должно совпадать с названием товара в базе данных.  
   Например, для товара «Молоко 3.2%» файл должен называться `Молоко 3.2%.png`.
3. Скрипт сам находит каждый товар по имени файла, копирует изображение  
   в `public/uploads/` и обновляет поле `imageUrl` в базе данных.

### Пошаговая инструкция

**Шаг 1.** Посмотрите, какие товары есть в БД:

```bash
npm run bulk-upload -- --list
```

Вы увидите два списка: товары с изображениями и товары без изображений  
с подсказкой, как назвать файл для каждого.

**Шаг 2.** Поместите изображения в папку `scripts/images/`:

```
scripts/images/Молоко 3.2%.png
scripts/images/Яблоки Голден.jpg
scripts/images/Куриная грудка.jpg
```

Поддерживаемые форматы: `.jpg`, `.jpeg`, `.png`, `.webp`, `.gif`

**Шаг 3.** Запустите загрузку:

```bash
npm run bulk-upload
```

Скрипт выведет прогресс и итоговую статистику.

### Все доступные флаги

| Команда                                        | Описание                                                          |
| ---------------------------------------------- | ----------------------------------------------------------------- |
| `npm run bulk-upload`                          | Загрузить изображения для товаров, у которых ещё нет картинки     |
| `npm run bulk-upload -- --force`               | Перезаписать изображения для **всех** товаров                     |
| `npm run bulk-upload -- --list`                | Показать список товаров и статус их изображений                   |
| `npm run bulk-upload -- --dry-run`             | Симуляция: показать что будет сделано, **без** реальных изменений |
| `npm run bulk-upload -- --category "Молочные"` | Загрузить только для товаров из указанной категории               |
| `npm run bulk-upload -- --help`                | Показать справку по всем флагам                                   |

## 8. Архитектура приложения

### Технологический стек

| Технология       | Версия | Назначение                                        |
| ---------------- | ------ | ------------------------------------------------- |
| **Next.js**      | 16     | React-фреймворк с файловым роутингом и API Routes |
| **React**        | 19     | Библиотека построения UI                          |
| **TypeScript**   | 5      | Статическая типизация                             |
| **Prisma ORM**   | 5      | Работа с базой данных, типобезопасные запросы     |
| **SQLite**       | —      | Встроенная реляционная БД, не требует сервера     |
| **Tailwind CSS** | 4      | Utility-first CSS-фреймворк                       |

### Схема базы данных

```
Category ──< Product >── OrderItem >── Order
```

- **Category** — категория товара (Фрукты, Молочные и т.д.)
- **Product** — товар с ценой, описанием, остатком и ссылкой на изображение
- **Order** — заказ с именем клиента, адресом, телефоном и статусом
- **OrderItem** — позиция заказа (товар + количество + цена на момент покупки)

### REST API

| Метод    | URL                 | Описание                                             |
| -------- | ------------------- | ---------------------------------------------------- |
| `GET`    | `/api/products`     | Список товаров (с фильтрами: category, sort, search) |
| `POST`   | `/api/products`     | Создание нового товара                               |
| `GET`    | `/api/products/:id` | Один товар по ID                                     |
| `PATCH`  | `/api/products/:id` | Обновление товара                                    |
| `DELETE` | `/api/products/:id` | Удаление товара                                      |
| `GET`    | `/api/categories`   | Список категорий                                     |
| `POST`   | `/api/upload`       | Загрузка изображения (multipart/form-data)           |

### Поток данных на главной странице

```
Пользователь вводит фильтры
        │
        ▼
  useFilters (хук)
  хранит: categoryId, sortBy, sortOrder, search, currentPage
        │
        ▼
  useProducts (хук)
  debounce 300 мс → GET /api/products?... → массив товаров
        │
        ▼
  page.tsx
  slice() → paginatedProducts (12 товаров на страницу)
        │
        ▼
  <ProductCard /> × 12
```

## 9. Устранение проблем

### `node -v` не работает

Node.js не установлен. Скачайте по ссылке: [https://nodejs.org/](https://nodejs.org/)

### Ошибка при `npm install`

Попробуйте очистить кэш и установить заново:

```bash
npm cache clean --force
npm install
```

### Ошибка базы данных при запуске

Сбросьте БД и заполните заново:

```bash
npm run db:reset
```

### Порт 4000 уже занят

Завершите процесс, занимающий порт 4000, или укажите другой порт:

```bash
# Для Windows: найти и завершить процесс
netstat -ano | findstr :4000
taskkill /PID <номер_процесса> /F
```

### Страница `/admin` не отображает товары

Убедитесь, что сервер запущен (`npm run dev`) и база данных заполнена (`npm run db:seed`).

### Изображения не отображаются после bulk-upload

Убедитесь, что файлы скопированы в `public/uploads/`:

```bash
# Просмотр папки uploads
dir public\uploads
```

Проверьте, что поле `imageUrl` обновилось в БД: запустите `npm run bulk-upload -- --list`.
